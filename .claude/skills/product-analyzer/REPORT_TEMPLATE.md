## 2. UI/UX 风格和质量描述

### 2.1 视觉风格
配色、字体、插画/图标风格、整体调性(企业级 / 玩具感 / 极简 / 拟物 …)。挂代表性截图。

### 2.2 信息密度与层级
首屏给到的信息量是否过载或太空、主要 CTA 是否一眼能找到、次要功能是否被合理藏起来。

### 2.3 交互流畅度
- 启动到首屏耗时(主观感受)
- 关键操作(创建任务 / 切换 tab / 打开设置)是否有明显延迟、动画掉帧、状态指示模糊
- 反馈是否及时(点击有无 hover/press 反馈、加载是否有 indicator)

### 2.4 文案质量
官网与应用内文案是否一致、有无机翻味、专业术语和情感表达的平衡。挂引用。

### 2.5 可访问性观察(肉眼可见的)
对比度是否足够、是否有键盘可达性问题、深色模式支持、字号是否可调。**注**:完整 a11y 验证需要专门工具与辅助技术评估,这里只记录直观观察。
