## 1. 详细的产品逻辑介绍

### 1.1 总定位

CC Switch 是一款跨平台桌面应用（v3.15.0），基于 MIT 协议开源，专为使用 AI 编程工具的开发者设计。它通过统一的图形界面，帮助用户管理 Claude Code、Claude Desktop、Codex、Gemini CLI、OpenCode、OpenClaw 和 Hermes Agent 等 AI 编程工具的 API 供应商配置，解决多供应商切换麻烦、配置分散难管理、无法监控用量等痛点。

### 1.3 各界面功能与评价

#### 1.3.1 官网首页

- **功能**：首屏展示产品名称、标语"统一管理你的 AI 编程工具工作流"、版本标签（v3.15.0 正式发布）、右侧应用界面预览图、CTA 按钮（免费下载、查看文档）。底部注明支持 macOS 12+、Windows 10+、Linux。
- **交互**：用户可通过导航栏切换首页/文档/攻略/更新日志/赞助商；可点击 CTA 按钮进入下载或文档流程。
- **评价**：首屏信息密度适中，产品定位清晰。右侧预览图直观展示了应用主界面（供应商列表），让用户一眼理解产品形态。但"免费下载"按钮链接至 404 页面，是一个明显的功能缺陷。
- **截图**：[01_web_homepage.png](screenshots/01_web_homepage.png)

#### 1.3.2 首页功能卡片区

- **功能**：以六宫格卡片形式展示核心功能：统一管理七大应用、自动故障转移、用量与额度可见、安全本地存储、MCP/Skills/会话、开源免费。
- **交互**：纯展示，无可交互元素。
- **评价**：卡片设计统一，每个功能配图标和简短描述，信息传递高效。"开源免费"卡片明确标注"基于 MIT 协议开源"，降低了用户试用门槛。
- **截图**：[02_web_features.png](screenshots/02_web_features.png)

#### 1.3.3 文档-软件介绍页

- **功能**：详细阐述产品定义、解决的三大痛点（多供应商切换麻烦、配置分散难管理、无法监控用量）。左侧导航栏展示文档结构：快速入门、供应商管理、扩展功能、本地路由与高可用、常见问题。
- **交互**：左侧导航可展开/收起，点击跳转对应章节。
- **评价**：文档结构清晰，痛点描述具体（如"Claude Code、Claude Desktop、Codex、Gemini、OpenCode、OpenClaw、Hermes 各有独立的配置文件，格式不同"），开发者用户容易产生共鸣。
- **截图**：[03_web_docs_intro.png](screenshots/03_web_docs_intro.png)

#### 1.3.4 文档-核心功能页

- **功能**：分"供应商管理"和"扩展功能"两大块展开。供应商管理包括：一键切换多个 API 供应商配置、支持预设模板、统一供应商功能跨应用共享配置、Claude Desktop 第三方供应商/直连模式与模型映射、用量查询与余额显示、端点速度测试。扩展功能包括：MCP 服务器（管理 Model Context Protocol 服务器）、Prompts（管理系统提示词预设）。
- **交互**：纯文本展示。
- **评价**：功能列表详细且具体，覆盖了开发者使用 AI 工具时的主要场景。"端点速度测试"是一个有差异化的功能，可帮助用户选择响应更快的供应商。
- **截图**：[04_web_docs_features.png](screenshots/04_web_docs_features.png)

## 2. UI/UX 风格和质量描述

### 2.1 视觉风格

整体采用浅色主题（米色/奶油色背景 `#f5f0e8`），搭配橙色/珊瑚色（`#e07a5f`）作为主色调，营造温暖、亲和的开发者工具氛围。Logo 为彩色放射状图案，辨识度高。应用界面预览图显示白色卡片式布局，供应商列表以卡片形式展示，每个供应商配有图标、名称、URL、用量统计。

### 2.2 信息密度与层级

首屏层次分明：Logo + 导航栏 → 版本标签 → 主标题 → 副标题 → CTA 按钮 → 应用预览图。主要 CTA（免费下载）使用橙色填充按钮，位于左下角，视觉权重足够。次要功能（查看文档）使用描边按钮，主次区分明确。功能卡片区六宫格布局均匀，每个卡片内图标+标题+描述三层信息，无过载感。

### 2.3 交互流畅度

网页加载速度正常，无明显的加载延迟。导航栏 hover 状态未观察到（截图无法 capture）。文档页左侧导航的展开/收起未实际操作验证。首页"免费下载"按钮点击后跳转至 404，存在功能缺陷。

### 2.4 文案质量

官网文案为地道中文，无机翻痕迹。产品标语"统一管理你的 AI 编程工具工作流"简洁有力。文档中痛点描述具体（"各有独立的配置文件，格式不同"），专业术语使用恰当（MCP、Provider、Schema 迁移等）。版本号标注"v3.15.0 正式发布"传达了产品的成熟度。

### 2.5 可访问性观察

首页文字与背景对比度充足，橙色按钮上的白色文字可读性良好。导航栏文字大小适中。右上角提供深色模式切换按钮（月亮图标），说明支持暗色主题。未观察到键盘导航相关的可见标识。

## 3. 官网描述

### 3.1 关键文案摘录

> "CC Switch 是一款跨平台桌面应用，专为使用 AI 编程工具的开发者设计，它帮助你统一管理 Claude Code、Claude Desktop、Codex、Gemini CLI、OpenCode、OpenClaw 和 Hermes 等配套应用的供应商配置。"
> —— 来源：文档-软件介绍

> "统一管理你的 AI 编程工具工作流"
> —— 来源：首页 H1 副标题

> "基于 MIT 协议开源，完全免费使用，社区驱动开发，欢迎贡献代码和反馈。"
> —— 来源：首页功能卡片-开源免费

### 3.2 核心卖点（官网视角）

1. **统一管理平台**：一个界面管理七大 AI 编程工具的供应商配置（文档-软件介绍）
2. **自动故障转移**：本地路由内置熔断器、健康监控，主 Provider 异常时自动切换（首页功能卡片）
3. **用量透明**：实时追踪请求、Token、缓存命中、成本和订阅额度（首页功能卡片）
4. **安全本地存储**：所有配置和 API Key 存储在本地 SQLite 数据库（首页功能卡片）
5. **开源免费**：MIT 协议开源，社区驱动（首页功能卡片、页脚）

## 4. 目标用户

基于官网用语与实际功能推断：

- **主要用户**：使用 AI 编程工具（Claude Code、Claude Desktop、Codex 等）的开发者
- **痛点场景**：同时使用多个 AI 工具、需要切换不同 API 供应商（官方/中转）、关注 API 用量和成本
- **技术门槛**：需要理解 API Key、Provider、MCP 等概念，面向有一定技术背景的开发者

---

## 5. 与同类产品对比

| 维度 | CC Switch | API 管理工具（如 One API） | 官方客户端 |
|---|---|---|---|
| 定位 | AI 编程工具专用配置管理 | 通用 API 代理/转发 | 单一工具配套 |
| 支持工具 | 7+ 种 AI 编程工具 | 通用 OpenAI 兼容 API | 仅自身 |
| 本地存储 | SQLite 本地数据库 | 通常服务端部署 | 本地 |
| 开源协议 | MIT | 多为 MIT/Apache | 闭源 |
| 用量监控 | 内置，支持多维度 | 服务端统计 | 依赖供应商 |

CC Switch 的差异化在于"面向 AI 编程工具生态"而非通用 API 管理，且完全本地运行、无需服务端部署。

## 6. 优劣势小结

| 维度 | 优势 | 劣势 |
|---|---|---|
| 产品逻辑 | 精准定位 AI 编程工具配置管理痛点；支持 7+ 工具统一配置；MIT 开源降低使用门槛 | 官网下载功能不可用；依赖 GitHub release 分发，对国内用户不够友好 |
| UI/UX | 界面简洁直观；功能卡片信息传递高效；文档结构清晰 | 官网存在 404 链接；部分导航链接无效 |
| 工程质量 | 跨平台支持（macOS/Windows/Linux）；本地 SQLite 存储保障安全；v3.15.0 版本说明持续迭代 | 官网维护存在疏漏；GitHub 被限流时无法获取安装包 |
